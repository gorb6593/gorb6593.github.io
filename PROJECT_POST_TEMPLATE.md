# Project Post Template

아래 템플릿을 복사해서 `_posts/YYYY-MM-DD-slug.md` 파일로 만들면 됩니다.

```md
---
layout: post
title: "프로젝트 제목"
author: "이해규"
tags: ["Product", "Case Study"]
---

한 줄 소개를 먼저 씁니다.
이 프로젝트를 왜 만들었는지, 무엇을 개선했는지 2~3문장으로 적습니다.

## 문제 상황

- 어떤 불편이 있었는지
- 왜 이 프로젝트가 필요했는지
- 누가 이 문제를 겪었는지

## 내가 한 일

### 1. 설계

- 어떤 기준으로 구조를 잡았는지
- 기술 선택 이유

### 2. 구현

- 실제로 만든 기능
- 중요하게 본 포인트

### 3. 운영 / 개선

- 배포 후 어떻게 개선했는지
- 사용자 반응이나 운영 이슈

## 결과

- 수치가 있으면 수치
- 없으면 사용자 경험 변화
- 배운 점

## 회고

다음에는 무엇을 더 개선할지,
이 프로젝트를 통해 무엇을 배웠는지 정리합니다.
```

예시 파일:

- `_posts/2024-04-20-icam-product-story.md`
- `_posts/2024-04-20-maeumgirok-product-story.md`
- `_posts/2024-04-20-customer-registration-flow-improvement.md`
