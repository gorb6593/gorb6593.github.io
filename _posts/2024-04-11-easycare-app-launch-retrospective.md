이 글은 이지케어 앱 신규개발 작업을 돌아보며, 레거시 앱을 현대화하고 실제 사용자 흐름을 개선하기 위해 어떤 점에 집중했는지 정리한 회고입니다.

## 배경

기존 앱은 오래된 구조와 제한된 기능 때문에 서비스 확장과 사용자 경험 개선에 한계가 있었습니다.

- 레거시 앱 구조로 유지보수 부담이 컸습니다.
- 앱 고유 기능을 충분히 활용하지 못했습니다.
- 웹뷰 기반 서비스 확장도 더 유연한 구성이 필요했습니다.

## 목표

- 레거시 Flutter 앱 현대화
- 푸시 알림 등 앱 고유 기능 활성화
- 웹뷰 서비스에 주·야간보호 기능 확장

## 내가 맡은 역할

- 앱 기능 개선에 필요한 백엔드 및 연동 구조 지원
- FCM 기반 푸시 알림 시스템 구축
- 기존 서비스 흐름과 앱 기능이 자연스럽게 연결되도록 정리

## 어떻게 진행했는가

### 1. 레거시 구조 정리

먼저 현재 앱이 무엇을 잘 못하고 있는지를 정리했습니다.

- 유지보수 포인트 파악
- 기능 확장 시 제약 요소 확인
- 사용자 입장에서 불편한 흐름 확인

### 2. 앱 기능 강화

단순히 기존 앱을 다시 만드는 것이 아니라, 모바일 앱다운 경험을 추가하는 것이 중요했습니다.

- FCM 기반 푸시 알림 시스템 구축
- 앱 고유 기능 활용 범위 확대
- 사용자가 더 빠르게 필요한 기능에 접근하도록 흐름 개선

### 3. 웹뷰 서비스 확장

웹뷰 형태의 기존 서비스도 단순 연결이 아니라, 앱 안에서 자연스럽게 이어지도록 보완했습니다.

- 주·야간보호 기능 추가
- 앱과 웹뷰 사이의 사용 흐름 정리

## 결과

- 레거시 앱 대비 이후 확장이 가능한 기반 마련
- 푸시 알림 기반 사용자 접점 강화
- 웹뷰 서비스 범위를 넓혀 실제 활용 기능 증가

## 아쉬웠던 점

- 앱/웹뷰 경계가 있는 구조라 완전히 일관된 경험을 주는 데 한계가 있었습니다.
- 이후에는 앱 전용 기능과 웹 기능의 분리를 더 명확히 설계하면 좋겠다고 느꼈습니다.

## 회고

앱 신규개발은 결국 “새로 만드는 것”보다 **기존 서비스 경험을 모바일 환경에 맞게 다시 설계하는 일**에 가까웠습니다.  
이번 경험을 통해 앱은 단순 기능 추가보다도, 사용자가 어디에서 끊기고 어디에서 다시 들어오는지를 기준으로 설계해야 한다는 점을 더 분명히 배웠습니다.
