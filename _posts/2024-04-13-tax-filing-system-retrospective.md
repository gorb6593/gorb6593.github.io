이 글은 리드넘버에서 진행한 **부가세/종합소득세 신고 시스템 고도화** 작업을 돌아보며, 왜 자동화가 중요했고 어떤 방식으로 신고 준비 기간을 줄일 수 있었는지 정리한 회고입니다.

## 배경

세무 신고 업무는 시즌성 트래픽과 반복 수작업이 함께 몰리는 대표적인 영역이었습니다.

- 신고 시즌이 되면 업무량이 급격히 증가
- 데이터 수집과 검증에 많은 시간이 소요
- 운영팀의 수작업 부담이 큼

## 목표

- 신고 준비에 필요한 데이터 수집 자동화
- 검증/전처리 프로세스 자동화
- 운영팀이 실제로 체감할 수 있는 시간 단축

## 내가 맡은 역할

- 외부 API 연동 구조 설계 및 개발
- Spring Batch 기반 검증/전처리 자동화
- 운영 프로세스 병목 분석과 개선

## 어떻게 진행했는가

### 1. 수작업 구간 분석

먼저 사람들이 시간을 많이 쓰는 구간을 파악했습니다.

- 어떤 데이터를 어디서 가져오는지
- 어떤 단계에서 검증과 보정이 필요한지
- 운영팀이 반복적으로 처리하는 일이 무엇인지

### 2. 데이터 수집 자동화

외부 연동을 통해 필요한 매출 데이터를 자동으로 가져오도록 구성했습니다.

- 홈택스 API 연동
- 배달앱 API 연동
- 데이터 수집 흐름의 일관성 확보

### 3. 검증과 전처리 자동화

수집된 데이터를 그대로 사용하는 것이 아니라, 실제 신고 준비에 바로 쓸 수 있는 수준으로 정리하는 것이 중요했습니다.

- Spring Batch 기반 검증
- 전처리 로직 자동화
- 예외 데이터 확인 포인트 분리

## 결과

- 신고 준비 기간 **75% 단축** (4주 → 1주)
- 운영팀 반복 업무 감소
- 신고 시즌 대응 속도와 안정성 향상

## 아쉬웠던 점

- 외부 API 품질과 데이터 예외 케이스에 따라 예상보다 세밀한 보정 로직이 많이 필요했습니다.
- 이후에는 예외 유형을 더 구조적으로 분류하면 운영 효율을 더 높일 수 있을 것 같습니다.

## 회고

이 프로젝트는 자동화가 단순한 효율 개선이 아니라, **시즌성 업무를 버틸 수 있는 운영 체력을 만드는 일**이라는 점을 보여준 사례였습니다.  
개발자는 기능만 만드는 것이 아니라, 조직의 반복 비용을 줄이는 구조를 설계해야 한다는 점을 다시 강하게 느꼈습니다.
